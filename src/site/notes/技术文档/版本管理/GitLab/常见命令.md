# gitlab-ctl

`gitlab-ctl <OPTIONS>`

**参数**
- `restart`
- `reconfigure`
# gitlab-backup
备份目录的所有者和组别必须是`git`, 可通过`chown`命令修改, 比如`eg. sudo chown -R git:git /var/opt/gitlab/backups`

备份`gitlab`数据到备份目录, 默认为`/var/opt/gitlab/backups`

自定义备份目录需要通过`gitlab_rails['backup_path']`指定
不包括配置文件
## 基本格式
`gitlab-backup <OPTIONS>`

**参数**
- `create`: 把本地`gitlab`数据打包放到指定路径
	- `eg. gitlab-backup create`: 会生成类似的文件`eg. 1782475410_2026_06_26_19.0.3_gitlab_backup.tar`
- `restore`: 从默认/自定义备份目录, 重载数据到当前`gitlab`中
	- `eg. gitlab-backup restore BACKUP=1782475410_2026_06_26_19.0.3`
	- `force=yes`: 表示恢复过程中出现的提示自动`yes`
		- `eg. gitlab-backup restore BACKUP=1782475410_2026_06_26_19.0.3 force=yes`

1. 使用git hook（钩子）

1. 步骤

1. 在.git/hooks/目录下创建commit-msg文件，并添加以下示例脚本

```shell
#!/bin/sh

COMMIT_MSG_FILE=$1
COMMIT_MSG=$(<$COMMIT_MSG_FILE)

# 检查提交消息是否符合规范
if ! echo "$COMMIT_MSG" | grep -q "^[A-Z]"; then
  echo "Error: Commit message must start with a capital letter."
  exit 1
fi

# 添加更多检查...
```

1. 确保脚本有执行权限

1. 作用：这个脚本会在每次提交前运行，如果提交消息不符合规范，提交将失败

1. 客户端钩子

1. 服务端钩子：主要是针对于git push操作前后事件的处理

1. Pre-Receive：每当有用户使用git push命令推送提交到仓库时，pre-receive钩子就会被触发

1. 注意

1. 该脚本不接受参数，但是推送的引用会按照下面的格式传递给脚本

1. 示例

```python
#!/usr/bin/env python​
import sys
import fileinput​
# Read in each ref that the user is trying to update
for line in fileinput.input():    
    print "pre-receive: Trying to push ref: %s" % line
​# Abort the push
# sys.exit(1)
```

1. Update:工作原理上同，但是会根据推送的多个引用分别被调用，即尝试推送四个分支，update会被执行四次

1. 注意：该钩子不需要从标准输入中读取数据，而是接收下面三个参数

1. 0：要更新的引用名称

1. 1：存储在引用中的旧的提交对象名称

1. 2：新的对象名称

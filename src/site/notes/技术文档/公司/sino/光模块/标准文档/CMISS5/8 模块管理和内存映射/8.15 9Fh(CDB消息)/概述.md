# Table 8-129 9Fh页概览(CDB 消息)
- 128-133: 命令消息头,  参考[[技术文档/公司/sino/光模块/标准文档/CMISS5/8 模块管理和内存映射/8.15 9Fh(CDB消息)/概述#Table 8-130 命令消息头\|#Table 8-130 命令消息头]]
	- 宿主机写命令消息头
- 134-135: 响应消息头
	- 模块写头信息添加到响应消息中
- 136-255: 本地负载(LPL)
	- 存储命令 和/或 响应消息负载的位置

# Table 8-130 命令消息头
## 128-129 CMD ID
- `U16`
- 指定`CDB`命令码用于执行一个`CDB`命令并写入这个字段, 也可以说是宿主机给模块发送了一个`CDB`消息
- 注
	- `Table 9-7`列出了所有命令和他们的`CMD ID`, 参考[[技术文档/公司/sino/光模块/标准文档/CMISS5/9 CDB命令使用/9.4 CDB特性和功能介绍/概述#Table 9-7 CDB特性和功能命令概览\|Table 9-7]]
	- 这个模块介绍了对消息传输来说实际的触发条件
	- 宿主机必须在写129之前或者连写包括129在内的操作时, 组成它的头部信息和负载
## 130-131 EPLLength(Extended Payload Length, 扩展负载)
- `U16`
- 用于指定存储在`EPL`的命令消息负载字节的长度
- 有效长度为0-2048
- 注: 这个`EPLLength`字段是被包含在`CdbChkCode`字段值的计算中的

## 132 LPLLength(Local Payload Length, 本地负载长度)
- `U8`
- 注: 上同
## 133 CdbChkCode(cdb校验码)
- `U8`
- `cdb`校验码是通过宿主机把字节`9Fh:128`到`9Fh:(136+LPLLength-1)`并且不包括字节`9Fh:133-135`作为一个整体计算出来的值
- `EPL`页的完整性校验(如果有)对于每个CMD是单独定义在第9章

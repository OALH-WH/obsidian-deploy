## 设备树移植步骤
### 确认当前配置
- 查看应用中的`build/primary/zephyr/zephyr.dts`
### 添加.overlay文件覆写设备树

主要通过`CMake`的变量`DTC_OVERLAY_FILE`包含用空格( ) 或者 分号(;)分隔的`.overlay`文件列表

下述几种设置它的方法:

- 在`cmake`构建命令行加上`-DDTC_OVERLAY_FILE="file1.overlay;file2.overlay"`
- 在`CMakeLists.txt`中, 在包含`boilerplate.cmake`文件之前, 用`set()`赋值变量
- 在应用中创建`boards/<BOARD>_<revision>.overlay`文件, 如果这个板子有要支持多个修订版本的需求, 参考 [Multiple board revisions](https://docs.zephyrproject.org/2.7.0/guides/porting/board_porting.html#porting-board-revisions). 如果`boards/<BOARD>.overlay`文件存在, 则`boards/<BOARD>_<revision>.overlay`文件会合并过去
- create a `boards/<BOARD>.overlay` file in the application folder, for the current board
- create a `<BOARD>.overlay` file in the application folder
- create an `app.overlay` file in the application folder

这个构建系统打印的所有设备树`.overlay`文件都能在配置阶段找到, 就像这样: 
```shell
-- Found devicetree overlay: .../some/file.overlay
```
### .overlay中修改设备树配置
#### 删除一个属性
```dts
// 删除一个属性, 在属性前面加上/delete-property/
&serial0 {
     /delete-property/ some-unwanted-property;
};
```

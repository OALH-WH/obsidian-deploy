# 🗺️ Zephyr 项目结构概览

下面这个表格可以帮你快速了解这三大组成部分的区别：

|组成部分|核心角色与含义|主要内容与作用|
|---|---|---|
|**`zephyr` (Zephyr内核)**|Zephyr RTOS的核心与灵魂。|存放Zephyr操作系统的核心代码，包括内核、驱动、子系统（如蓝牙、USB）及硬件支持。这是你应用的基础运行平台[](https://docs.panchip.com/pan1080dk-doc/0.8.0/pan1080dk.pdf#84#55)。|
|**`bootloader` (引导程序)**|系统的启动“第一棒”，主要用于安全启动和固件升级。|默认使用开源、安全的**MCUboot**。它负责验证App固件（`slot0_partition`）的签名与完整性[](https://docs.panchip.com/pan1080dk-doc/latest/docs-zdk/04_dev_guides/zephyr_bootloader_guidance.html)，支持安全的固件升级（`slot1_partition`存储更新包）[](https://docs.panchip.com/pan1080dk-doc/latest/docs-zdk/04_dev_guides/zephyr_bootloader_guidance.html)和回滚。|
|**`modules` (外部模块)**|可复用的“插件”库，用于集成第三方或独立维护的代码。|由`west.yml`清单文件管理，作为独立Git仓库存在。每个模块都有`zephyr/module.yml`文件。典型模块包括MCU厂商的**HAL（硬件抽象层）**、文件系统、加密库等。|

# Zephyr 启动流程
## 1. Zephyr C运行环境准备阶段

**deps/zephyr/arch/arm/core/cortex_m/reset.S, 架构相关示例**

初始化栈帧, 赋值`msp`寄存器
```assembly
/* 这里是要得到栈帧的最高地址, z_main_stack是一个数组首地址 */
ldr r0, =z_main_stack + CONFIG_MAIN_STACK_SIZE

/* 把得到的栈帧最高地址赋值给msp, 在arm架构下, 后续压栈的时候, msp寄存器的值就从这个地址往低地址减少 */
msr msp, r0
```



最后, 跳转到`z_prep_c`函数执行

**deps/zephyr/arch/arm/core/cortex_m/prep_c.c, 架构相关示例**

执行`z_prep_c`函数, 完成下述工作:

- 中断向量表重定位, `relocate_vector_table()`
- 清零`.bbs`段, `arch_bss_zero()`
- 从`Flash`中把数据段的内容拷贝到`RAM`的`.data`段中, `arch_data_copy()`
- 中断初始化, `z_arm_interrupt_init()`
- `...`
- 跳转到`z_cstart`函数执行

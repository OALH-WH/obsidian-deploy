# 背景
## clash处理连接请求, 大致步骤
1. 收到一个连接请求，目标为域名（例如gitlab.sino-telecom.com）或者IP
2. 解析DNS发生在不同阶段
	1. 如果使用`redir-host`模式，Clash会在匹配域名规则之前先解析域名（如果规则是域名规则，则使用配置的DNS服务器解析），然后根据解析得到的IP进行IP规则匹配（如果有的话）。
	2. 如果使用`fake-ip`模式，Clash会返回一个假的IP地址（如198.18.0.x），当请求发送到这个假IP时，Clash会再根据域名进行匹配和转发。
3. Clash首先检查规则，如果规则中有匹配该域名的规则（如DOMAIN-SUFFIX,gitlab.sino-telecom.com,DIRECT），则使用该规则（DIRECT）进行转发。
4. 但是，在转发之前需要将域名解析为IP地址。对于DIRECT规则，Clash会使用系统DNS或配置的DNS服务器进行解析。
5. 如果DNS解析返回了正确的IP地址，则连接将直接发送到该IP（不经过代理）。
6. 如果DNS解析返回了错误的IP，或者解析超时，则连接可能失败。

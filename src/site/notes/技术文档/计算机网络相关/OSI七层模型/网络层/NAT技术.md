1. 全称：Network Address Traslation
2. 原理
	- 虚拟机内部：虚拟机会获取一个私有的 IP 地址（例如 192.168.x.x），并且通过宿主机的网络接口与外部通信。
	- 虚拟化软件的路由器：虚拟化软件在宿主机上创建一个虚拟路由器，该路由器会将虚拟机的请求从私有网络转发到宿主机的网络接口。它会使用 **NAT（网络地址转换）** 来将虚拟机的私有 IP 地址映射为宿主机的公共 IP 地址。
	- 外部响应：外部网络（例如互联网）对虚拟机的响应会首先发送到宿主机，然后宿主机会根据 NAT 转换表将数据包正确地转发到相应的虚拟机。
	- 虚拟机对外通信：虚拟机能够访问外部网络（如访问互联网），但外部网络无法直接访问虚拟机。虚拟机的通信是由宿主机代理的，因此外部只能通过宿主机的 IP 地址与虚拟机进行间接通信。

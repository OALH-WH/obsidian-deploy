#### 主状态码

| 标志      | 全称                        | 含义                                                                                                                |
| ------- | ------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| **`R`** | **Running / Runnable**    | **正在运行** 或 **可运行**（在运行队列中等待CPU时间片）。进程要么正在CPU上执行，要么随时准备执行。                                                         |
| **`S`** | **Interruptible Sleep**   | **可中断睡眠**。进程在等待某个事件完成（如I/O操作、信号）。可以被信号（如 `SIGKILL`, `SIGTERM`）立即唤醒。这是最常见的等待状态。                                    |
| **`D`** | **Uninterruptible Sleep** | **不可中断睡眠**。进程在等待通常与硬件相关的I/O操作（如读磁盘、网络通信），不能被任何信号中断。看到这个状态通常意味着I/O繁忙或有I/O瓶颈。**杀不掉此类进程，只能等I/O完成或重启系统。**             |
| **`T`** | **Stopped**               | **已停止**。进程被作业控制信号（如按 `Ctrl-Z` 发送的 `SIGTSTP`）或调试器（如gdb）暂停。                                                         |
| **`t`** | **Tracing Stop**          | **跟踪停止**。进程正被另一个进程（如调试器）跟踪或控制，这也是一种特殊的停止状态。(Linux 2.6.33+后引入，更精确地区分调试停止)                                          |
| **`Z`** | **Zombie / Defunct**      | **僵尸状态 / 已死**。进程已完成执行（调用了 `exit`），但其父进程还未调用 `wait()` 系统调用来获取其退出状态。进程表（PCB）条目仍被保留，但进程本身已无内存、CPU等资源。僵尸进程过多会耗尽PID资源。 |
| **`X`** | **Dead**                  | **已死亡**。进程正在被销毁，或已完全消失。这是一个瞬态，在 `ps` 列表中几乎看不到。                                                                    |
#### 附加标志
| 标志      | 含义                                                           |
| ------- | ------------------------------------------------------------ |
| **`<`** | **高优先级进程**。进程的 `nice` 值为负，获得了人为提升的调度优先级。                     |
| **`N`** | **低优先级进程**。进程的 `nice` 值为正，拥有了人为降低的调度优先级。                     |
| **`L`** | **有内存页被锁定**。部分或全部内存页被锁定在物理内存中，不允许换出到Swap分区。通常是实时进程或需要高性能的进程。 |
| **`s`** | **会话领导者**。进程是该会话的领导者。在Shell中，你启动的第一个命令就是这个会话的领导者。            |
| **`l`** | **多线程进程**。进程包含多个线程，Linux将其实现为轻量级进程。                          |
| **`+`** | **位于前台进程组**。该进程属于其控制终端的前台进程组，可以直接与用户交互。                      |

### `COMMAND`
- **方括号 `[]`**：表示该进程是一个**内核线程**。它不是由用户程序启动的，而是内核自身创建的，用于执行内核级的后台任务。这些进程不读取用户命令，因此 `CMD` 列只是内核分配给它的一个内部名称

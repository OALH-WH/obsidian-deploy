# 原理
## 流程
-  **命令触发**：你执行 `mount -t ext4 /dev/sdb1 /mnt/data`。`-t ext4` 明确告诉内核：“请使用 `ext4` 文件系统来挂载这个设备。”
-  **内核查找处理程序**：
    - 内核的VFS层会去查找一个名为 `ext4` 的**文件系统类型**。这个类型在内核中对应一个 `struct file_system_type` 数据结构，其中包含了该文件系统的关键方法（如 `mount` 方法）。
    - 如果 `ext4` 被**静态编译**进内核，这个结构早已在内核数据区，直接可用。
    - 如果 `ext4` 是**模块**且尚未加载，内核就找不到这个结构。
-  **自动加载模块（关键步骤）**：
    - 当内核找不到 `ext4` 时，它会通过 `request_module(“ext4”)` 机制，**向用户空间发送一个加载模块的请求**。
    - 这个请求通常由 `modprobe` 工具处理。`modprobe` 会去标准模块目录（如 `/lib/modules/$(uname -r)/kernel/fs/ext4/`）下查找 `ext4.ko` 文件。
    - `modprobe` 调用系统调用，将 `ext4.ko` **加载到内核空间**并执行其初始化函数。这个函数会向内核**注册** `ext4` 这个 `file_system_type` 结构。
-  **完成挂载**
    - 现在内核有了 `ext4` 的处理能力。VFS会调用 `ext4` 文件系统模块提供的 `.mount` 方法。
    - `ext4` 模块通过驱动读取 `/dev/sdb1` 上的**超级块**，验证其格式，并在内存中建立相关的数据结构（`super_block`, `inode` 等）。
    - 最后，内核将这个新挂载的文件系统关联到你指定的目录 `/mnt/data`。此后，所有对 `/mnt/data` 的操作，都会经由VFS路由到 `ext4` 模块的代码去处理。

# 流程
1. **格式化 (`mkfs.ext4 /dev/sdb1`)）**：
    - 用户空间的 `mkfs.ext4` 工具，会**调用内核中的 `ext4` 模块代码**。
    - `ext4` 模块按照自己的“蓝图”（数据结构定义），在块设备 `/dev/sdb1` 的空白扇区上，**写入一套初始的、标准的元数据**（超级块、空的inode表等）。
    - **此时，我们说“块设备 `/dev/sdb1` 上有了一个 ext4 文件系统”。** 这指的是**物理结构**。
2. **挂载 (`mount /dev/sdb1 /mnt`)）**：
    - 当执行挂载命令时，内核需要**理解** `/dev/sdb1` 上的数据结构。
    - 如果 `ext4` 模块尚未加载，内核会动态加载 `ext4.ko`。
    - 加载后，`ext4` 模块的代码被激活，它**读取并解析** `/dev/sdb1` 上的那些元数据，在内存中建立起管理结构（`struct super_block`, `inode` 缓存等）。
    - **此时，我们说“内核通过 `ext4` 文件系统模块挂载了该设备”。** 这指的是**内核的能力和运行状态**。
3. **文件操作 (`echo “hello” > /mnt/test.txt`)）**
    - 应用程序的写操作通过VFS，调用 `ext4` 模块的写函数。
    - `ext4` 模块根据内存中的元数据，决定将数据写在块设备的哪些空闲块上，并更新内存中的inode信息。
    - 最终，这些更新被写回到块设备上相应的扇区（如更新inode表、位图）。
    - **内核模块是“操作者”，块设备上的结构是“被操作和记录的仓库”。**

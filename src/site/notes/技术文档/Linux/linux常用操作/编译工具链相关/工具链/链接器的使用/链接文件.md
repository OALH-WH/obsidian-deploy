# 链接作用
将所有目标文件的段, 合并到一个文件中, 并对这些段进行内存布局
# 链接文件格式
其文件名后缀支持如下格式:
- `***.ld`
# 链接语法
## 块
### MEMORY
这个是可选的, 主要用来描述支持的内存区域, 不定义的话表示所有段都在一块内存区域, 并且从`0x0`地址开始

示例格式如下:
```ld
MEMORY  
{  
FLASH (rx)      : ORIGIN = 0x8000000, LENGTH = 128K  
RAM (xrw)      : ORIGIN = 0x20000000, LENGTH = 32K  
}
```
### SECTIONS
用于定义目标文件中各个段，包括名称、存储位置、对齐方式、内容，并且可以确定其访问权限。

示例格式如下:
```ld
SECTIONS 
{  
// 将可执行代码段（.text 段）存放到 RAM 区域，对齐到 4 字节边界
.text >RAM : ALIGN(4) 
// 将全局变量段（.data 段）存放到 FLASH 区域，对齐到 4 字节边界
.data >FLASH : ALIGN(4) 
// 将堆（heap）段存放到 RAM 区域，对齐到 8 字节边界
.heap >RAM : ALIGN(8)
}
```
另外, **链接脚本中的内存存放顺序是从上到下**
#### 语句, 片段
示例解释: 
```ld
SECTIONS  
{  
. = 0x10000;  // 将地址指针设为0x10000，表示.text段从地址0x10000开始  
.text : { *(.text) }  // 表示将输入文件的.text段复制到输出文件的.text段中  
. = 0x8000000;        //将地址指针设为0x8000000，表示.data段从地址0x8000000开始  
.data : { *(.data) }  
.bss : { *(.bss) }  
}
```


## 命令
### ENTRY
将某个入口点函数指定为起始地址，如果没有指定`ENTRY`，链接器会默认使用第一个可执行`section`作为程序入口点

| ENTRY 的作用 | 在 ELF 文件头写一个"入口地址"         |
| --------- | -------------------------- |
| 硬件启动      | 看向量表 0x08000004，不关心 ENTRY  |
| GDB/调试器启动 | 可能读 ENTRY 来设置 PC           |
| 不写 ENTRY  | 链接器默认用 .text 段开头，通常能跑但不够精确 |

最佳实践: ENTRY(你的Reset_Handler)

1. 作用

2. 程序运行时，将各个段的内容，加载到指定的CPU地址空间的某块地址上，并进行内存布局，启动文件决定如何搬运各个段的内容

3. 基本概念（VMA和LMA都是CPU内存空间的绝对地址）

4. 注意

5. AT() 明确给出 LMA；> RAM 给出 VMA。

6. linux和单片机的区别

7. 对于单片机，存储的地址为flash在cpu地址空间映射的地址

8. 对于linux程序，加载地址和链接地址无实际用处，实际由mmu完成重定位

9. VMA：链接地址，程序实际运行的地址，CPU取值/取地址用到的地址

10. 示例：如果SDRAM映射到CPU地址空间的地址为0x3000_0000的话，而链接脚本有如下配置，则.data段的VMA就是0x3000_0000

```cpp
.data 0x30000000 : AT(0x800) 
```

1. LMA：加载地址，程序实际被加载到物理存储器中的地址

1. 链接器把一个或多个输入文件合成一个输出文件

1. 输入文件：目标文件或者链接脚本文件

1. 输出文件：目标文件或者可执行文件

1. 目标文件的每个段至少包含两个信息：名字和大小，大部分段还包含与其相关联的一块数据，为contents（段内容）。一个段可以被声明为loadable和allocatable

1. loadable：输出文件运行时，相应的段内容将被加载到地址空间中，如.text段和.data段

1. allocatable：在输出文件运行时, 在进程地址空间中空出大小同段指定大小的部分. 某些情况下, 这块内存必须被置零。如**.bss**段。

1. 简单示例

```
SECTIONS

. = 0×10000;
 
.text :     { *(.text) }

.bss :     { *(.bss) }

```

1. 示例详解

1. . = 0x10000：把定位器符号置为0x10000(若不指定，则该符号的初始值为0)

1. .text：{*（.text）}:将所有的输入文件的.text段合并成一个.text段，该段的地址由定位器符号的值指定，即0x10000

1. . = 0×8000000 ：把定位器符号置为0×8000000

1. .data : { *(.data) } : 将所有输入文件的.data 段合并成一个.data 段, 该段的地址被置为0×8000000.

1. .bss : { *(.bss) } : 将所有输入文件的.bss 段合并成一个.bss 段，该段的地址被置为0×8000000+.data 段的大小.

1.  注意：链接器每次读完一个段描述后，将定位器符号的值增加该段的大小，但是没有考虑对齐约束

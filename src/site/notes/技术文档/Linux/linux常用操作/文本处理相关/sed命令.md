# 背景
- 非交互式, 基于行的流文本编辑器
- 多行模式（Multi-line Pattern Space
{ #34f3c4}

	- 当 **模式空间（Pattern Space）中同时包含多于一行输入**（以 `\n` 分隔）时，sed 处于 **多行模式（Multi-line Pattern Space）**。
	- 触发条件
		- `command`: N...
## 工作流程
### 基本工作流程
- 以循环的方式工作
	- 读取一行输入到**模式空间(Pattern Space)**
	- 按顺序执行**脚本**中的命令
	- 默认将**模式空间**输出
	- 清空模式空间, 进入下一行
- 除非使用 `d, n, q, p, D`等控制命令改变流程
---
# 语法规则
## 脚本基本格式
- `[address[,address]] command [arguments]`
	- `address`
		- `N`: 表示第N行, `eg. 5`
		- `$`: 表示最后一行
		- `/REGEX/`: 正则表达式(regular expression)匹配的行, 默认一行匹配一次
		- `address!`: 表示地址否定, 即不匹配该行时执行 `command`
		- 示例
			- `1,10d`: 删除1到10行的内容
			- `/BEGIN/,/END/p`: 打印 `BEGIN`第一次匹配的行到 `END`第一次匹配的行
	- command
		- 行控制命令
			- `d`: 删除模式空间的内容, 并立即进入下一轮 `cycle`
			- `D`: 删除模式空间第一行的内容, 此命令一般在**多行模式**下使用, 参考[[技术文档/Linux/linux常用操作/文本处理相关/sed命令#^34f3c4\|#^34f3c4]]
			- `n`: 输出当前模式空间内容并清空, 开启新一轮 `cycle`
			- `N`: 将下一行追加到当前模式空间, 触发多行模式
			- `q`
			- `Q`
		- 输出控制命令
			- `p`: 小写
			- `P`: 大写
			- `=`
			- `l`
# 常见参数
- `-n`: 静默模式, 不匹配的行不输出
- `-E`: 启用正则扩展语法
- `-i <FILE>`: 直接根据脚本修改文件
	- 不指定时默认修改结果输出到标准输出
- `-e <multiple script>`: 每一行执行多个脚本, 可重复使用
	- `eg. sed "" [-e "" -e "" ...] FILE`
- `-f <FILE.sed>`: 从文件读取脚本
	- 脚本基本格式
```shell
script1
script2
...
```

## A. 行控制类命令（Flow Control）

| 命令          | 专业说明                 |
| ----------- | -------------------- |
| `d`         | 删除模式空间并立即开始下一轮 cycle |
| `D`         | 删除模式空间第一行（多行模式）      |
| `n`         | 输出当前行并读取下一行          |
| `N`         | 读取下一行并追加到模式空间        |
| `q`         | 立即退出 sed             |
| `Q`         | 立即退出，不输出当前模式空间       |
| `b [label]` | 无条件分支跳转              |
| `t [label]` | 条件跳转（基于替换成功）         |
| `T [label]` | 条件跳转（基于替换失败）         |
| `:`         | 定义标签                 |

|命令|说明|
|---|---|
|`p`|打印模式空间|
|`P`|打印模式空间第一行|
|`=`|输出当前行号|
|`l`|列表输出（显示不可见字符）|

# 六、正则表达式规则（Regular Expression Rules）

## 1. 基础正则（BRE，默认）

|符号|含义|
|---|---|
|`.`|任意字符|
|`*`|0 次或多次|
|`^`|行首|
|`$`|行尾|
|`[]`|字符类|
|`\{m,n\}`|重复次数|
|`\(...\)`|捕获组|
|`\1`|反向引用|

---

## 2. 扩展正则（ERE）

`sed -E 's/(abc)+/X/'`

|特性|说明|
|---|---|
|`()`|无需转义|
|`+`|1 次或多次|
|`?`|0 或 1|
|`|`|

# 八、sed 执行顺序规则（Critical Semantics）

1. **命令按书写顺序执行**
    
2. **地址先匹配，后执行命令**
    
3. `d` / `n` 会立即终止当前 cycle
    
4. `t` / `T` 依赖 **最近一次 s 命令状态**
    
5. 默认输出发生在 **cycle 结束**
    

---

# 九、sed 的典型“高级特性”

- 多行模式（Multi-line Pattern Space）
    
- 状态驱动流编辑（Branch + Hold Space）
    
- 类 DSL 脚本能力
    
- 单遍扫描（Single-pass）

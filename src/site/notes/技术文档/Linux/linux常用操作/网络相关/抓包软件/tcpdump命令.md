# 背景
- 抓包的时间点是在经过防火墙过滤之前
# 基本格式
- `tcpdump [OPTIONS] [PROTO] [DIR] [TYPE]`
## 可选参数
**显示控制相关**
- `-n`：以数字形式显示地址，而不是尝试解析为主机名
	- 注意：如果没有配置可达的DNS，不加-n可能导致命令阻塞等待超时，现象类似卡死，不能立即输出网络包
- `-A`: 以ASCII格式显示数据包内容
- `--text`: 以文本格式显示数据包内容
- `-v`: 显示更详细的数据包信息
- `-X`：以十六进制和`ASCII`格式显示数据包
- `-tttt`: 显示时间戳
- 
**捕获控制相关**
- `-i <interface>`: 指定网卡接口, 不指定默认从`eth0`捕获数据
	- `interface`: 指定具体的网卡接口, `eg. eth0 lo wlan0...`
	- `-i any`: 捕获所有网卡接口上的数据包
- `-c <datagram num>`: 指定捕获的数据包数量
- `-s <datagram size>`: 指定捕获的数据包大小，并不是通过数据包大小过滤，而是截断捕获的数据包到指定的
	- `-s 0`: 表示捕获完整数据包
- `-w <filename>`： 将捕获的数据包保存到文件中，文件后缀为`.pcap`,相同文件会覆盖之前保存的数据

**其他**
- `-r <filename>`：从文件中读取数据包
- `-q`：以更安静的方式执行命令
- `-D`:列出所有支持的网卡接口
## 位置参数
### 逻辑组合
- `and`
	- `tcpdump 'host 192.168.1.100 and not port 22'`
- `not`
- `or`
### PROTO
#### **网络层**
- `tcp`
- `udp`
- `icmp`
#### **数据链路层**
 ##### `arp`
 **字段解析**
- `who-has ... tell ...`: 表示成功发送了一个`arp`请求, 大致意思为: 谁有...告诉..., 下述示例本机`ip`为`192.168.0.200`
	- `eg. 18:08:57.901628 ARP, Request who-has 192.168.0.200 tell 192.168.0.200, length 44`: 表示发送了一个`arp`请求的内容为: 谁有`192.168.0.1`的物理地址, 告诉`192.168.0.200`
	- 
 
### DIR
- `src`: 表示过滤数据包的源信息
- `dst`: 表示过滤数据包的目标信息
### TYPE
- `host <ip addr>`: 过滤指定`ip`, `eg. host 192.168.1.2`
- `net <>`
- `port <NUM>`: 过滤指定端口, `eg. port 8090`
- `portrange <NUM1-NUM2>`: 过滤指定端口范围, `eg. portrange 8090-11111`

# 字段解析
- <系统时间> <源主机.端口> > <目标主机.端口> <数据包参数>
## Flag
- `[S]`：SYN，表示开始连接
- `[S.]`：表示SYN-ACK报文
- `[.]`：表示ACK
- `[P]`：表示PSH，推送数据
- `[P.]`:表示PSH，并带有延迟返回的ACK数据，一般出现这种情况还有可能是为了节省包数
- `[F]`：表示FIN，结束连接
- `[R]`：表示RST，重置连接，端口不可用的典型情况，一般分为服务没开（本地测试端口试试）和防火墙筛选（查看防火墙策略）掉了两种情况
- `[DF]`: 表示不分片，当DF为0时，允许分片， divide fragment
- `[FP.]`：FIN、PUSH、ACK

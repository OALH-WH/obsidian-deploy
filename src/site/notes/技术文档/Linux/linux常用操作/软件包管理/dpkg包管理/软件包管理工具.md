1. dpkg-buildpackage(打包工具)

1. 常用示例

1. dpkg-buildpackage -us -uc //包结构根目录执行

1. dpkg（低级包管理工具）：负责单个.deb软件包的安装卸载，不处理软件包的依赖关系

1. 常用示例

1. dpkg -i <package.deb>: 安装一个.deb文件

1. dpkg -r <package>: 卸载已经安装的包，只卸载包本身，不会处理配置文件

1. dpkg -l :列出已安装的包

1. dpkg -S <file>: 查找哪个包包含指定的文件

1. dpkg -c <deb package file>: 查看包内所有文件结构

1. dpkg -b <package dir name>:  对写好的包文件打包 //注意, 要求配置目录名为DEBIAN, 而不是debian

1. apt（高级包管理工具）

1. 常用示例

1. apt update：更新包管理器的包源索引

1. apt upgrade：升级系统中所有可以升级的包	

1. apt search <package>：搜索包

1. apt show <package>: 查看包的详细信息

1. sudo apt-get install --fix-broken：修复因缺失依赖软件包而损坏的包

1. apt build-dep <package>: 重新编译软件包???

1. apt-cache

1. 常用示例

1. apt-cache policy debhelper //查看软件包的已安装版本, 可以安装的版本, 软件源来自哪等信息

1. dh $@表示使用Deb Helper构建

1. override_dh_auto_build里写构建软件包的命令

1. override_dh_auto_install里写安装软件包的命令

1. override_dh_auto_clean里写执行清理操作的命令

1. override_dh_installsystemd里写服务安装规则

1. 

```cpp
#!/usr/bin/make -f

%:
	dh $@

override_dh_auto_build:
	# 在这里添加构建软件包的命令
	qmake
	make

override_dh_auto_install:
	# 在这里添加安装软件包的命令
	make INSTALL_ROOT=$(CURDIR)/debian/tmp install

override_dh_auto_clean:
	# 在这里添加清理操作的命令
	make distclean
 
override_dh_installsystemd:
    dh_installsystemd --name=hostcfgd # 指定安装的服务名称, eg. hostcfgd.service
```

1. control //包信息

```cpp
# content
Package: AppName         # 软件包名
version: 1.0             # 版本号 
Section: Application     # 软件类别, eg. Utils, Net, Library...
Priority: optional       # 软件对于系统重要程度, eg. required, standard, optional, extra
Architecture: armhf      # 软件包架构, eg. amd64, armhf, i386
Maintainer: ...          # 软件维护人员
Description: ...         # 描述
```

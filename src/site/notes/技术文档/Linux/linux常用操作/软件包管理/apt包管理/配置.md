# 软件镜像源
- `/etc/apt/sources.list`: 镜像源文件
## 常见问题
### 添加软件源会检查软件源的签名来保证安全, 如果没有验证密钥, 则会认为这个源不安全并拒绝使用
- 解决方案
```shell
# 如果是docker软件源的密钥丢失, 从docker官网获取公钥并加入系统可信密钥目录
curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
```
## 基本语法格式
```
deb [选项] URI 发行版 组件1 [组件2 ...]
deb-src [选项] URI 发行版 组件1 [组件2 ...]
```
## 含义解析
- `deb/deb-src`: 包类型, 即二进制/源代码
- `[选项]`: 可选参数
	- `eg. [arch=amd64]`
	- `eg. [check-valid-until=no]`: 忽略发布日期检查
	- ...
- `URL`: 镜像源地址
- 发行版: 发行版代号, 即 `codename`, 参考[[发行版本相关#`lsb_release`命令]]
	- `eg. bullseye`
	- `eg. kali-rolling`
- 组件: 软件包分类
	- `main`: 自由软件
	- `contrib`: 依赖于non-free的自由软件
	- `non-free`

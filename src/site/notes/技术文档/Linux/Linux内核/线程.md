# 线程分离
- **资源自动回收**：当一个分离的线程终止时，系统会自动回收其所有资源（如栈内存），无需其他线程调用 `pthread_join()
- **不可再合并**：一旦线程被分离，就不能再调用 `pthread_join()` 等待它结束，也无法重新设为可合并（joinable）状态
- **不影响进程终止**：如果主线程退出, 即`main`函数调用了`return`（或进程调用 `exit()`），所有线程（包括分离的）都会被强制终止，分离状态并不能让线程脱离进程的生命周期
- **父线程无关性**: 如果分离了子线程, 那么父线程的生命周期结束不会导致子线程退出, 只有主线程退出才会导致它退出.

# 僵尸线程
- **产生原因**: 即没有分离子线程, 也没有等待子线程, 子线程执行结束之后, 线程ID、栈、寄存器上下文等它占用的资源不会自动释放, 即会导致内存泄漏
- **后果**
	- 内存泄漏
	- 线程无法创建
	- 调试困难

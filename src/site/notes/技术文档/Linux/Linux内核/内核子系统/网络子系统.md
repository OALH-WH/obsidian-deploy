# 网络协议栈
大概分为以下几个层级:
- 套接口层(又叫协议无关接口层)
- 传输层
- 网络层
- 邻居子系统
- 网络设备接口
## 套接口层
- 提供系统调用接口的内核实现
- 关联协议族和协议类型
## 传输层
- 提供端到端的可靠/不可靠传输
## 网络层
- 负责主机到主机的通信, 路由选择, 分片和重组, 协议IP
## 邻居子系统
- **地址解析**：将网络层 IP 地址转换为链路层物理地址（MAC）。
    - IPv4：通过 **ARP**（地址解析协议）。
    - IPv6：通过 **ND**（邻居发现协议，使用 ICMPv6）。 
- **邻居缓存**：维护一个邻居表（`struct neighbour`），缓存 IP → MAC 的映射及邻居状态（INCOMPLETE、REACHABLE、STALE 等）。
- **请求队列**：当需要发送 IP 包但目标 MAC 未知时，临时挂起该包，发送 ARP/ND 请求，收到响应后重新发送挂起的包。
- **完成通知**：解析成功后，回调网络层的继续发送函数；解析失败则向上层返回错误。
- **其他**：代理 ARP、免费 ARP（检测地址冲突）、邻居可达性探测等。

**对上层**：接收网络层的输出请求（`neigh_output()`），决定是直接发送（已有邻居条目）还是触发解析流程。  
**对下层**：解析成功后，将填写好 MAC 地址的 skb 交给网络设备接口层。

# 背景
## vmlinux、Image，zImage、uImage 的区别
- `vmlinux`
	- 是编译出来的最原始的内核文件，是未压缩的
	- 构建完成后, 存放于**根目录**
- `Image`
	- Linux 内核镜像文件，但是 Image 仅包含可执行的二进制数据。Image 就是使 用 objcopy 取消掉 vmlinux 中的一些其他信息，比如符号表什么的
	- 构建完成后, 存放于 `./arch/<ARCH>/boot`
- `zImage`: 内核编译后生成的压缩镜像文件
	- 使用`gzip`压缩, 通常用于`x86`架构的引导
	- 构建完成后, 存放于 `./arch/<ARCH>/boot`
- `uImage`: 在`zImage`的基础上加了`64`字节头部信息
	- `U-Boot`专用的一种内核镜像格式
	- 这个头部信息包括: 加载地址, 入口地址, 映像名称, 映像大小等
	- 用于 `U-Boot`引导 `Linux`内核
	- 构建完成后, 存放于 `./arch/<ARCH>/boot`
# 构建配置
## `makefile`变量配置
### `ARCH`
- 架构类型, 例如arm
- 赋值类型为条件赋值, 参考[[make语法#4.2 `?=`（条件赋值）]]
### `CROSS_COMPILE`
- 交叉编译器工具链前缀, 例如`arm-none-linux-gnueabihf-`
- 赋值类型为条件赋值
### `LOADADDR`
- 指定构建好的内核加载地址
- 注意
	- 该变量生效的`makefile`不在根目录, 配置参考下述方法 
		- 命令行配置: `make LOADADDR=<ADDR>`
		- `makefile`配置如下
			- `override LOADADDR=<ADDR> `
			- `export LOADADDR`
# 构建目标
## `makefile`目标
### 注意
- 编译目标如`uImage, dtbs`这种目标, 需要 `LINUX-SOURCE-DIRECTORY/.config`的存在, 可通过手动配置`Kconfig`或者加载默认配置如`make XXX_defconfig`来填充`.config`
### 常见目标

#### gtags
生成静态索引, 生成文件有:
- `GTAGS`
- `GRTAGS`
- `GSYMS`
#### cscope
把整个内核源码全部扫描一遍，建立一个"谁在哪定义、谁调了谁"的交叉引用数据库，让你能像点鼠标一样在代码间跳来跳去
#### `modules_prepare`
- 给外部模块使用
- 写out-of-tree module, 使用这个
#### `prepare`
- 把内核源码变成可被C预处理器和IDE理解的源码
- 它不编译内核, 但会生成所有语法分析必须的头文件和偏移信息, 以及宏定义
- 在源码树的drivers下, 编写module, 使用这个
#### `dtbs`
- 编译设备树
- 需要内核的`KConfig`配置里选择了编译哪个设备树, 比如
	- `CONFIG_ARCH_STM32=y`: 表示编译`STM32`芯片厂商的设备树文件
- 编译好的设备树位置
	- 参考[[技术文档/Linux/Linux内核/源码结构/arch/arm/boot/dts/设备树文件\|设备树文件]]
- 设备树源码存放位置
	- 参考[[技术文档/Linux/Linux内核/源码结构/arch/arm/boot/dts/设备树文件\|设备树文件]]
- 最终设备树的结构会体现在设备起来之后`/proc/dvice-tree`目录下, 参考[[技术文档/Linux/常见目录和文件作用#**/proc**\|常见目录和文件作用#**/proc**]]
#### `uImage`
- 编译 `U-Boot`专用格式的Linux内核, 参考[[#`uImage`和`zImage`的关系]]
- 必须配置 `LOADADDR`, 并且需要和`U-Boot`引导 `Linux`内核时指定的加载地址一致, 参考[[#`LOADADDR` 用于指定加载地址]]
#### `distclean`
- 清理编译产生的全部内容, 一般比 `clean`清理的干净
#### `XXX_defconfig`
- 使用不同平台的默认配置, 参考[[结构简述#`configs` 保存不同平台的默认配置文件, 例如xxx_defconfig]]
- 会在根目录下生成一个`.config`文件, 用来存储当前配置

# KBuild Makefile 文档

## 概念

KBuild Makefile 是专为编译 Linux 内核而设计的 Makefile 语法格式。

## 语法

### 编译模块

使用以下命令编译内核模块：

```bash

make -C <kernel-dir> M=<module-dir> modules
```

#### module-dir

模块目标文件所在的目录。

#### kernel-dir

- **主机环境**：通常为 `/lib/modules/$(uname -r)/build`（这是一个指向内核源码根目录的软链接）。
    
- **交叉编译环境**：根据具体配置设定。
    

### obj-y 和 obj-m
- `obj-y` 或 `obj-m` 的选择基于内核配置文件 `.config` 中的设置。
    - `y` 表示将目标编译进内核镜像（built-in）。
    - `m` 表示将目标编译为可单独加载的内核模块。
#### obj-y
- `obj-y` 中列出的目标文件会被合并到 `built-in.o`，最终链接到内核镜像中。
### 多个源文件编译为目标文件

当模块由多个源文件组成时，使用以下变量指定依赖对象：

- `<module>-objs`
    
- `<module>-y`
    

其中 `<module>` 对应不带 `.o` 后缀的目标文件名。

#### `<module>-objs`

示例：

```makefile

# drivers/isdn/i4l/Makefile
obj-$(CONFIG_ISDN) += isdn.o
isdn-objs := isdn_net_lib.o i
```

#### `<module>-y`

示例：

```makefile

# fs/ext2/Makefile
obj-$(CONFIG_EXT2_FS) += ext2.o
ext2-y := balloc.o bitmap.o
ext2-$(CONFIG_EXT2_FS_XATTR) += xattr.o
```

**说明**：使用 `<module>-y` 可以根据配置条件灵活地添加或排除特定源文件，实现条件编译。

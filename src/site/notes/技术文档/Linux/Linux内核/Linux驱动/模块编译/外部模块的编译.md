# 基本步骤
- 执行`Linux`源码根目录下的`Makefile`, 编译目标设为`modules`, 并通过`M`环境变量指定**要编译的驱动项目根目录下的`Makefile`所在路径**
	- 示例 
		- `eg. make -C LINUX-SOURCE-DIR M=$(pwd) modules`: 在项目根目录下执行
- 编写驱动项目根目录`Makefile`中的内容
	- 配置**编译出来的驱动模块文件名称**, 下述变量含义参考[[技术文档/Linux/Linux内核/Linux驱动/KBuild Makefile系统/使用指南\|使用指南]]
		- `obj-m`: 指定编译出来的目标文件名, 即也是最终的驱动模块文件名称, `eg. obj-m := hello.o`
		- `<MODULE_NAME>-objs`: 指定依赖哪些源文件, `eg. hello-objs := main.o tools.o common.o[ ...]`

## 单源文件 `Makefile`编写
- `KERNELRELEASE`作用参考[[技术文档/Linux/Linux内核/内核构建/内核构建系统的实现详解/根目录Makefile#`KERNELRELEASE`|KERNELRELEASE]]
- `MAKE`的值参考[[make语法#`MAKE`|MAKE]]
```Makefile
ifneq ($KERNELRELEASE,)
	obj-m := hello.o
	hello-objs := main.o
else
	KERNELDIR ?= /linux-source-dir
	PWD := $(shell pwd)
default:
	$(MAKE) -C $KERNELDIR M=$PWD modules
endif
```

# 背景
- 文件系统时一种物理数据结果和组织格式, 必须持久化存储在某个地方(eg. 硬盘, SSD...)
- linux内核中的文件系统位于驱动层之上, VFS之下的可模块化(.ko)的代码结构和逻辑
- linux内核中的文件系统模块和块设备上的文件系统区别以及联系
	- 区别
		- linux内核中的文件系统模块相当于拥有建筑蓝图, 并无实际结构, 块设备驱动为施工队
		- 块设备上的文件系统相当于建筑实例, 块设备上存在实际的文件系统结构
	- 联系
		- linux内核中的文件系统模块用于读取, 管理, 操作格式化为对应文件系统的块设备, 即内核文件系统模块是文件系统的**解析器**和**控制器**

- 是一个基于内存的虚拟的根文件系统, 用于挂载真正的文件系统
- 通常由引导程序(u-boot)和Linux内核一起启动

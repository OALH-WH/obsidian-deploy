## 1. 传统 Unix IPC

- **管道（Pipe）**
    
    - **匿名管道**：用于有亲缘关系（父子进程）之间的单向通信，通过 `pipe()` 创建。
        
    - **命名管道（FIFO）**：通过文件系统路径名访问，可用于无亲缘关系的进程间通信，使用 `mkfifo` 创建。
        
- **信号（Signal）**
    
    - 异步通知机制，用于进程间发送简单事件通知（如终止、挂起等）。信号量有限，无法传递复杂数据。

## 2. System V IPC

- **消息队列（Message Queue）**
    
    - 消息以队列形式存储，进程可以发送和接收带有类型的消息。通过 `msgget`、`msgsnd`、`msgrcv` 操作，内核持久化。
        
- **共享内存（Shared Memory）**
    
    - 多个进程映射同一块物理内存，是最快的 IPC 方式。需配合同步机制（如信号量）使用。相关调用：`shmget`、`shmat`、`shmdt`。
        
- **信号量（Semaphore）**
    
    - 用于进程间同步，支持计数信号量。通常与共享内存配合使用。相关调用：`semget`、`semop`。

- 线程安全, 多线程情况下不需要额外加锁

## 4. 套接字（Socket）

- **Unix 域套接字**
    
    - 用于同一主机上的进程通信，比网络套接字更高效（不经过协议栈）。支持流式（SOCK_STREAM）和数据报（SOCK_DGRAM）。
        
- **网络套接字**
    
    - 支持跨主机通信（如 TCP/UDP），也可用于本地（通过 loopback 地址）。

## 5. 其他常用 IPC

- **文件（File）**
    
    - 通过读写共享文件交换数据，需处理锁和同步问题，效率较低但简单。
        
- **内存映射文件（Memory-mapped File）**
    
    - 通过 `mmap` 将文件映射到进程地址空间，多个进程可共享同一映射区域，相当于共享内存的文件版本。
        
- **D-Bus**
    
    - 高级 IPC 框架，用于桌面应用程序（如 GNOME、KDE）之间的通信，基于消息总线，支持服务注册和信号。
        
- **ZeroMQ / nanomsg**
    
    - 基于消息队列的网络库，封装了多种传输方式（如 TCP、进程内、IPC），提供更高级的通信模式（发布-订阅、请求-响应等）。

## 6. 进程间同步相关

- **互斥锁、条件变量（通过共享内存实现）**
    
    - 如果进程共享同一块内存，可以使用 pthread 的同步原语（需设置共享属性）。
        
- **文件锁（flock / fcntl）**
    
    - 通过对文件加锁实现进程间的互斥访问。

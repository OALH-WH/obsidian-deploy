# 背景
- 在使用 CMake 进行交叉编译时，核心思想是**保持 `CMakeLists.txt` 与平台无关**，而通过一个独立的**工具链文件（toolchain file）**来指定交叉编译的工具、目标系统和环境。这样，同一个 `CMakeLists.txt` 既可用于本地编译，也可用于交叉编译，无需修改。


# 配置
```cmake
# 指定目标系统名称，对于交叉编译至关重要

set(CMAKE_SYSTEM_NAME "Linux") # 或 "Windows", "Android", "RaspberryPI" 等

  

# 指定C和C++编译器的路径

set(CMAKE_C_COMPILER "/path/to/your/gcc")

set(CMAKE_CXX_COMPILER "/path/to/your/g++")

  

# (可选) 指定根路径，帮助CMake查找目标系统的库和头文件

set(CMAKE_FIND_ROOT_PATH "/path/to/target/sysroot")
```

# 功能
## 跳转APP功能
本质是模拟上电启动时的代码执行流程, 主要分为下述三个步骤:
- 清理不必要的外设资源, 比如关闭`systick`, 清除中断, 清除缓存, 关闭冗余的外设时钟, 减少功耗
- 初始化进入复位处理函数之前需要做的事情, 可以参考汇编启动代码
	- 比如内核库`CMSIS`中的`ST`的`startup_stm32h750xx.S`
	- 
- 跳转到`APP`复位处理函数
- 在`APP`中修改驱动, 防止覆盖`bootloader`中初始化好的外设, 并设置向量表偏移位置
### 功能定位总览

| 资源类别           | 归属方      | 举例                                       | XIP场景处理原则          |
| -------------- | -------- | ---------------------------------------- | ------------------ |
| ARM内核资源        | ARM定义    | SysTick, NVIC, SCB, MPU, FPU, CONTROL寄存器 | 必须清理，与XIP无关        |
| 通用MCU外设        | 厂商定义     | UART, Timer, ADC, GPIO, DMA, SPI, I2C    | 必须清理，与XIP无关        |
| **QSPI外设及依赖链** | **厂商定义** | **QSPI控制器, QSPI引脚, QSPI时钟源, QSPI映射控制器**  | **绝对保护，原封不动**      |
| 系统时钟树          | 厂商定义     | RCC/CGU/时钟控制模块                           | 只能关闭非QSPI分支，不能整体复位 |

**核心原则：清理所有ARM内核资源 + 清理厂商非QSPI外设，死保QSPI依赖链。**

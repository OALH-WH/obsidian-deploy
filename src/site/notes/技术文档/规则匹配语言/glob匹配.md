# 背景
- 主要用于**文件路径和名称匹配**（如命令行 `ls *.txt`、Git 的 `.gitignore`）。
    
- 简单直观，适合人类快速书写。
    
- 通常只处理文件名，不跨行匹配。

---
# 语法
## 2. 通配符与特殊符号

|符号|含义|示例|
|---|---|---|
|`*`|匹配任意长度的字符串（不包括路径分隔符 `/`）|`*.log` 匹配所有 `.log` 文件|
|`?`|匹配任意单个字符（不包括 `/`）|`?.txt` 匹配 `a.txt`，不匹配 `ab.txt`|
|`[abc]`|匹配方括号内的任意单个字符|`[0-9].log` 匹配 `1.log`、`2.log` 等|
|`**`|匹配任意层级的目录（包括零层）|`a/**/b` 匹配 `a/b`、`a/x/b`、`a/x/y/b`|
|`/`|路径分隔符，也可用于指定相对根目录或目录匹配|见下文|
|`!`|取反，重新包含被忽略的文件|`!important.log` 不忽略 `important.log`|
|`\`|转义特殊字符|`\#file` 匹配 `#file` 文件|

## 3. 路径匹配规则

### 3.1 斜杠 `/` 的特殊语义

- **以 `/` 开头**：模式只匹配仓库根目录下的文件或目录。  
    例如 `/build` 匹配根目录下的 `build` 文件/目录，但不匹配 `src/build`。
    
- **以 `/` 结尾**：模式只匹配目录（及其内容）。  
    例如 `build/` 忽略所有名为 `build` 的目录（及其内部所有文件），但不匹配名为 `build` 的普通文件。
    
- **包含 `/` 在中间**：模式匹配相对于 `.gitignore` 所在目录的路径。  
    例如 `docs/*.pdf` 仅忽略当前目录下 `docs/` 中的 PDF 文件。
    

### 3.2 默认相对性

- 如果不以 `/` 开头，模式匹配仓库中任何位置的文件/目录（不跨 `/` 限制）。  
    例如 `*.log` 忽略所有目录下的 `.log` 文件。
    

### 3.3 取反规则

- 以 `!` 开头的模式表示**重新包含**之前被忽略的文件。
    
- 取反对已经明确被父目录忽略的文件无效，必须重新指定其父目录不被忽略（如果有必要）。
    
- 例如：
    
    text
    
    *.log
    !important.log
    
    会忽略所有 `.log` 文件，但保留 `important.log`。
    

### 3.4 双星号 `**`

- `**` 可以匹配任意路径段，常用于匹配深层目录。
    
    - `**/foo` 匹配任何名为 `foo` 的文件或目录（任意层级）。
        
    - `abc/**` 匹配 `abc` 目录下的所有内容。
        
    - `a/**/b` 匹配 `a/b`、`a/x/b`、`a/x/y/b` 等。
        

---

## 4. 匹配顺序与优先级

- Git 按行从上到下依次应用规则，**后出现的规则不会覆盖前面的规则**，但取反规则 `!` 可以重新包含。
    
- 如果文件同时被忽略和包含规则匹配，以**最后出现的匹配规则**为准？实际上，取反规则优先于忽略规则，但需要确保取反规则出现在对应的忽略规则**之后**才生效。
    
- 建议将取反规则放在忽略规则之后。

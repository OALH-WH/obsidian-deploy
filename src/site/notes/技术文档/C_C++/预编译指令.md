# 指令
## `#include`
- 包含头文件
## `#pragma`
执行一些编译器特有指令
- `once`: 防止头文件重复包含 
# 宏
## 背景
- `STR`: 字符串字面量, 源码中带有`"`的字符
- `SIGN`: 标记, 源码中不带`"`的字符
- `ARG`: 宏参
## 连接符
- `#[ARG | SIGN]`: 用于把宏参数或者标记转换为字符串字面量(**不会进一步解析宏定义**)
- `##[ARG | SIGN]`: 用于和前项**拼接**宏参数/标记(**不会进一步解析宏定义**)
```cpp
#define CONCAT(a, b) a##b
CONCAT(my, var) // -> myvar
```
**注意**
```cpp
// 当嵌套宏调用时, 使用##, 需要两层, 保证宏定义的展开
#define CONCAT(a, b) a##_##b
#define FUNC_NAME(driver, suffix) CONCAT(driver, suffix)
```
## 基本格式
- `#define MACRO [...]`
- `#define MACRO(ARG1, ARG2) [...]`
- `#define MACRO`
## 可变参数宏格式
- `#define LOG_D(...)  printf("DEBUG: " __VA_ARGS__)`
	- 当调用 `LOG_D("value = %d", 42);` 时，`...` 匹配 `"value = %d", 42`，而 `__VA_ARGS__` 会被替换为 `"value = %d", 42`，最终展开为 `printf("DEBUG: " "value = %d", 42);`

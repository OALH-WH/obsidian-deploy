# 常用 C 字符串函数
## 复制
### strcpy(dest, src)
```c
char dest[10];
strcpy(dest, "Hello");  // 简单但危险！
```

| 优点   | 缺点               |
| ---- | ---------------- |
| 简单直观 | 不检查缓冲区大小，极易缓冲区溢出 |
| 兼容性好 | 源字符串必须 \0 结尾     |

### strncpy(dest, src, n)
 ```c
   char dest[10];
   strncpy(dest, "Hello World", 9);  // 最多复制9字节
   dest[9] = '\0';  // 手动确保结尾！
 ```

| 优点          | 缺点                                               |
| ----------- | ------------------------------------------------ |
| 可限制复制长度     | 不保证 \0 结尾（需手动添加）                                 |
| 比 strcpy 安全 | 可能复制多余数据（性能问题）<br>比如`src`少于`dest`, 会用`\0`填充多余的空间 |
### sprintf

| 优点          | 缺点           |
| ----------- | ------------ |
| 自动在结尾添加`\0` | 可能出现缓冲区溢出的风险 |

### snprintf
```c
char dest[10];
snprintf(dest, "hello world, hello world", 10); //会截断, 只保留9个char, 并在最后一个元素上写入`\0`
```

| 优点           | 缺点  |
| ------------ | --- |
| 防止缓冲区溢出      |     |
| 会在结尾自动添加`\0` |     |

 🔷 1. 字符串复制

 ### strcpy(dest, src)

 ```c
   char dest[10];
   strcpy(dest, "Hello");  // 简单但危险！
 ```

 ┌──────────┬──────────────────────────────────┐
 │ 优点     │ 缺点                             │
 ├──────────┼──────────────────────────────────┤
 │ 简单直观 │ 不检查缓冲区大小，极易缓冲区溢出 │

 │ 兼容性好 │ 源字符串必须 \0 结尾             │
 └──────────┴──────────────────────────────────┘

 ### strncpy(dest, src, n) ⭐ 推荐

 ```c
   char dest[10];
   strncpy(dest, "Hello World", 9);  // 最多复制9字节
   dest[9] = '\0';  // 手动确保结尾！
 ```

 ┌────────────────┬──────────────────────────────┐
 │ 优点           │ 缺点                         │
 ├────────────────┼──────────────────────────────┤
 │ 可限制复制长度 │ 不保证 \0 结尾（需手动添加） │
 ├────────────────┼──────────────────────────────┤
 │ 比 strcpy 安全 │ 可能复制多余数据（性能问题） │
 └────────────────┴──────────────────────────────┘

 ### memcpy(dest, src, n)

 ```c
   char dest[10];
   memcpy(dest, "Hello", 5);  // 精确复制5字节
 ```

 ┌────────────────┬─────────────────────────────────┐
 │ 优点           │ 缺点                            │
 ├────────────────┼─────────────────────────────────┤
 │ 精确控制字节数 │ 不处理 \0，不是专门的字符串函数 │
 ├────────────────┼─────────────────────────────────┤
 │ 可用于任意内存 │ 重叠内存时行为未定义            │
 └────────────────┴─────────────────────────────────┘

 ### memmove(dest, src, n) ⭐ 推荐（内存重叠时）

 ```c
   char str[20] = "Hello World";
   memmove(str+6, str, 5);  // 安全处理重叠："Hello Hello"
 ```

 ┌──────────────────┬────────────────┐
 │ 优点             │ 缺点           │
 ├──────────────────┼────────────────┤
 │ 安全处理重叠内存 │ 比 memcpy 略慢 │
 └──────────────────┴────────────────┘

 ⚠️ 现代最佳实践

 ```c
   // ❌ 危险写法
   strcpy(buf, input);

   // ✅ 安全写法
   strncpy(buf, input, sizeof(buf) - 1);
   buf[sizeof(buf) - 1] = '\0';

   // ✅✅ 更推荐（C11 Annex K 或 BSD）
   strlcpy(buf, input, sizeof(buf));  // 自动处理 \0
 ```

 核心原则：
 1. 永远不要信任输入长度
 2. 始终指定最大操作长度
 3. 手动确保 \0 结尾
 4. 重叠内存用 memmove

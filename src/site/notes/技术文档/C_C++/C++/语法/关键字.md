### **常量与类型限定符**

| 关键字                     | 核心作用与精确定义                                     | 关键细节与补充                                                                                                                                                                                                                            |
| ----------------------- | --------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`const`**             | 定义常量性或只读性。                                    | 1. **指针**：`const`在`*`**左**表指针**本身**为常量（`int* const p`）；在`*`**右**表指向数据为常量（`const int* p`）。  <br>2. **成员函数**：`void func() const;` 表明该函数不修改对象状态（可访问`mutable`成员）。  <br>3. **默认内部链接**：在文件作用域的`const`变量默认具有**内部链接性**（类似`static`），不担心重定义。 |
| **`constexpr`** (C++11) | 声明对象或函数可在**编译期**求值，是`const`的**超集**。           | 1. **变量**：必须是编译期可知的常量（如 `constexpr int size = 10;`）。  <br>2. **函数**：当传入编译期常量参数时，能返回编译期结果；否则可作为普通函数在运行期使用。  <br>3. **C++14/17**：放松了对函数体的限制，允许循环、局部变量等。                                                                              |
| **`consteval`** (C++20) | 声明**立即函数**，**必须**在编译期求值。                      | 用于强制编译期计算，比`constexpr`更严格。调用`consteval`函数若无法在编译期求值则报错。                                                                                                                                                                             |
| **`volatile`**          | 阻止编译器对该变量的读写进行优化，指示其值可能被“外部因素”（如硬件、多线程）意外更改。  | 1. 告诉编译器每次使用都必须从内存重新读取，修改后必须写回内存。  <br>2. **与`const`结合**：`const volatile`表示一个只读但可能被外部改变的硬件寄存器。                                                                                                                                     |
| **`mutable`**           | 突破`const`限制，允许在`const`成员函数或`const`对象中修改该成员变量。 | 典型用途：用于实现**内部缓存**（Cache）、**访问计数**、**调试日志**等与对象逻辑“常量性”无关的技术状态。                                                                                                                                                                      |

### **存储类说明符与作用域**

|关键字|核心作用|关键细节与补充|
|---|---|---|
|**`static`**|**1. 静态存储期**：变量在程序生命周期内存在。  <br>**2. 控制链接与可见性**。|1. **局部变量**：在函数内保持值不变，只初始化一次。  <br>2. **全局变量/函数**：使其具有**内部链接**，仅在当前文件可见。  <br>3. **类成员**：属于类本身而非对象，所有对象共享。需在类外单独定义。|
|**`extern`**|**声明**一个变量或函数在**其他翻译单元中定义**，具有**外部链接**。|1. 用于跨文件访问全局变量或函数。  <br>2. `extern “C”` 用于C++调用C语言编写的函数，抑制名称修饰。|
|**`thread_local`** (C++11)|指示变量具有**线程存储期**，每个线程有该变量的独立实例。|用于需要在线程内共享，但线程间隔离的数据，是线程安全的存储方式之一。|
|**`register`** (已弃用)|**提示**编译器将变量存储在寄存器中。C++17起弃用。|现代编译器优化能力极强，此提示通常被忽略。|

### **内联、命名空间与别名**

|关键字|核心作用|
|---|---|
|**`inline`**|**1. 函数内联建议**（编译器可忽略）。  <br>**2. 允许在多个翻译单元中定义同一函数/变量（C++17）**而不违反ODR，是头文件中定义函数/全局变量的关键。|
|**`namespace`**|划分标识符的作用域，避免命名冲突。支持嵌套和匿名命名空间。|
|**`using`**|**1. 类型别名**：`using NewType = OldType;` (比`typedef`更清晰)。  <br>**2. 命名空间引入**：`using std::cout;` 或 `using namespace std;`（慎用）。|

### **面向对象编程 (OOP)**

|关键字|核心作用|关键细节与补充|
|---|---|---|
|**`class`/`struct`/`union`**|定义类、结构体、联合体。`class`成员默认`private`；`struct`默认`public`。|`union`：所有成员共享内存，大小由最大成员决定。|
|**访问控制**|`public`（公开）、`protected`（保护，派生类可访问）、`private`（私有，仅本类可访问）。|是封装性的核心体现。|
|**`virtual`**|声明**虚函数**，实现**动态多态**（运行时绑定）。|1. **析构函数**：基类析构函数**必须**为`virtual`，否则通过基类指针删除派生类对象会导致**派生部分未析构**。  <br>2. **成本**：引入虚函数表（vtable）和虚表指针（vptr），有轻微空间和时间开销。|
|**`override`** (C++11)|**显式标记**此函数**重写**了基类的虚函数。|1. **强制编译器检查**：函数签名（返回类型、函数名、参数列表）必须与基类虚函数严格匹配，否则报错。  <br>2. 提升代码可读性和安全性。|
|**`final`** (C++11)|**1. 修饰类**：禁止该类被继承。  <br>**2. 修饰虚函数**：禁止该函数在派生类中被进一步重写。|与`override`可同时使用，表示“此函数重写了基类虚函数，且是最终版本”。|
|**`explicit`**|禁止构造函数的**隐式类型转换**。|1. 也适用于**转换运算符**（C++11起），防止意外的隐式转换。  <br>2. 提高代码明确性，避免歧义。|
|**`friend`**|授权其他类或函数访问本类的`private`和`protected`成员。|1. **友元关系不可传递、不可继承**。  <br>2. 常用于**重载运算符**（如`<<`，`>>`）和某些需要紧密协作的工具函数。|
|**`this`**|指向当前对象的**指针**。在非静态成员函数内部可用。|用于返回对象自身引用（`return *this;`），或解决成员变量与参数同名的问题。|

### **内存管理**

|关键字|核心作用|关键细节与补充|
|---|---|---|
|**`new`/`delete`**|进行**动态内存分配/释放**，并调用构造函数/析构函数。|1. **成对使用**：`new` 对应 `delete`；`new[]` 对应 `delete[]`。  <br>2. **定位new**：在指定内存位置构造对象。|
|**`malloc`/`free`**|C语言库函数，只进行**原始内存的分配/释放**，不处理对象构造与析构。|在C++中与`new`/`delete`混用是**未定义行为**。|
|**`sizeof`**|返回类型或对象的**字节大小**。|对类、数组、指针等使用时的结果需特别注意（如指针的大小固定）。|
|**`alignof`**/`alignas` (C++11)|查询/指定类型的**内存对齐要求**。|用于优化内存布局或满足特定硬件/协议要求。|

### **模板与泛型编程**

|关键字|核心作用|
|---|---|
|**`template`**|声明模板，是泛型编程的基础。|
|**`typename`**|**1. 在模板中声明类型参数**（与`class`在此等价）。  <br>**2. 在模板内部指示一个依赖名称是类型**（如 `typename T::iterator`）。|
|**`constexpr`**|使模板能在编译期进行计算和实例化。|

### **异常处理**

|关键字|核心作用|关键细节与补充|
|---|---|---|
|**`try`/`catch`/`throw`**|实现异常处理机制。`try`监控代码块，`catch`捕获异常，`throw`抛出异常。|1. 异常开销较大，不应用于常规控制流。  <br>2. 遵循“异常安全”原则。|
|**`noexcept`** (C++11)|**1. 声明函数不抛出任何异常**。  <br>**2. 运算符**：`noexcept(expr)`判断表达式是否可能抛异常。|1. 为编译器提供优化机会（如避免生成展开代码）。  <br>2. 移动构造函数等关键函数标记为`noexcept`可使标准库容器操作更高效。|

### **运算符与类型转换**

|关键字|核心作用|
|---|---|
|**`operator`**|用于**重载运算符**，赋予用户自定义类型与内置类型相似的操作语义。|
|**`dynamic_cast`**|**动态类型转换**，用于具有继承关系的多态类型，转换失败返回`nullptr`（指针）或抛异常（引用）。依赖RTTI。|
|**`static_cast`**|**静态类型转换**，用于相关类型间明确的转换（如非`const`转`const`、派生类转基类、`void*`转其他指针等）。编译期检查。|
|**`reinterpret_cast`**|**低层重新解释**，将内存按另一种类型解释，极度危险，不保证可移植性。|
|**`const_cast`**|移除或添加`const`和`volatile`限定符。常用于调用旧式API。|

### **其他重要关键字**

| 关键字                         | 核心作用                                  |
| --------------------------- | ------------------------------------- |
| **`enum class`** (C++11)    | **强类型枚举**，解决了传统`enum`的命名污染和隐式转换问题。    |
| **`nullptr`** (C++11)       | 空指针常量，类型安全，取代`NULL`宏和字面量`0`。          |
| **`typeid`**                | 返回`std::type_info`对象，用于查询类型信息（RTTI）。  |
| **`static_assert`** (C++11) | **编译期断言**，用于在编译时检查条件，不满足则停止编译并给出错误信息。 |
| **`auto`** (C++11)          | **自动类型推导**，让编译器根据初始化式推导变量类型。          |
| **`decltype`** (C++11)      | 查询表达式的**声明类型**，常用于模板和泛型代码中。           |

## 常见通用寄存器
### R15, PC寄存器
- 存储下一条要执行的指令的地址
- 当CPU每执行完一条指令硬件自动实现自增(+2/+4, 取决于指令长度), 从而能够顺序执行下一条指令
- 可以通过指令显式修改
- **中断/异常**：当发生中断时，硬件自动将 PC（以及一些其他寄存器）压入栈中，然后将 PC 设置为中断向量表里对应的中断服务函数地址；中断返回时，硬件从栈中恢复 PC，继续之前的执行。
### R13, SP寄存器
SP 在 ARM Cortex-M 中是 **R13**，分为 **MSP**（主堆栈指针，复位后默认使用，也在中断处理程序中使用）和 **PSP**（进程堆栈指针，在操作系统任务中使用）。它指向**当前栈的顶部**（栈是向下增长的，即压栈时 SP 减小，出栈时 SP 增大）。

SP 的值决定了**栈在内存中的位置**。栈主要用于：
- **保存函数返回地址**（如果函数内还需要调用其他函数，则会先保存 LR）。
- **保存局部变量**（C 编译器自动将局部变量分配到栈上）。
- **保存寄存器现场**（在中断或函数调用前，如果某些寄存器会被修改，需要先压栈保存，恢复时再出栈）。

# 指令集

## 背景
ARM 是一家只设计不生产的公司，它定义了一套又一套的**架构规范**，然后将这些规范授权给其他芯片厂商（如你手头芯片的制造商ST、NXP、高通等）。不同系列的处理器遵循了不同的规范：
- **Cortex-M 系列**：遵循的是 **ARMv6-M、ARMv7-M、ARMv8-M** 等架构规范[](https://cloud.tencent.cn/developer/article/1837579?from=15425)[](https://wiki.lceda.cn/zh-hans/design-production/electronic-selection/chip-architecture.html)。这套规范是为**微控制器（MCU）** 量身定制的，目标是做一颗强大又好用的“控制大脑”。因此，它强调极快的中断响应（nS级别）、确定性的行为、低功耗和精简的指令，让程序能紧密地操控硬件外设[](https://m.gec-edu.org/gsnews/show/576520.html)。
- **Cortex-A 系列**：遵循的是 **ARMv7-A、ARMv8-A、ARMv9-A** 等架构规范[](https://www.arm.com/en/architecture/learn-the-architecture/a-profile)。这套规范是为**应用处理器（MPU）** 量身定制的，目标是做一颗能运行复杂操作系统的“计算心脏”。因此，它必须包含内存管理单元（MMU，Memory Management Unit）来支持多进程和虚拟内存，并加入更强的多媒体处理指令（如NEON、SVE）来应对音视频、AI等计算任务[](https://www.arm.com/en/architecture/learn-the-architecture/a-profile)[](http://m.eepw.com.cn/article/201404/235983.html)。
## 标志位
```assembly
TST R0, #0x0F

; Z (Zero)      = 1 如果 (R0 & 0x0F) == 0
; N (Negative)  = 1 如果结果的 bit31 = 1
; C (Carry)     = 通常不变（某些情况可能会变）
; V (Overflow)  = 不变

; 所以可以这样用：
TST R0, #0x80000000
BMI negative                ; N=1 → bit31 = 1
```
## 常见伪指令
### `.global`
声明全局符号，使其可以在其他模块中访问
**格式**
`.global <name>`
### .section
创建一个新的段, 并指定段名, 段名允许包含`.`, 常见段名有
- `.text`: 表示代码段, 都是指令
- `.data`: 表示数据段, 都是已经初始化的数据
- `.bss`: 表示未初始化的数据段
- `.rodata`: 表示只读数据段
#### ELF Version
**格式**
`.section name [, "flags"[, @type][,flag_specific_arguments]]`

**参数**
- `flag`
### `.align`
用于设置数据的对齐方式，以确保在内存中按照特定边界对齐。
### `.macro`
定义宏，可以在代码中多次调用以简化代码
### `.word`
相当于占位符, 表示在当前地址上存放指定的`4-bytes`的数据, 这个数据可以是在链接脚本中定义, 它只规定数据在这个段的存放位置, 至于数据是什么, 它不会管
### `.weak`
表示该标号可以在其他地方重定义，即声明一个弱符号（weak signal），可以在其他地方声明强符号（strong signal），在处理弱符号和强符号时会优先使用强符号
### `.type`
给符号分类, 符号默认没有类型, 但可以通过`.type`伪指令标记分类, 比如下述几个类型:
- 函数
- 数据对象
- `...`
#### ELF Version

**格式**
`.type <name> , <type description>`
**参数**
- `name`: 符号名
- `type description`: 标记的类型
	- 格式支持有:
		- `.type <name> STT_<TYPE_IN_UPPER_CASE>`
		- `.type <name>,#<type>`
		- `.type <name>,@<type>`
		- `.type <name>,%<type>`
		- `.type <name>,"<type>"`
	- 类型有:
		- `function`: 把符号标记为一个函数

## 常见汇编指令

### 运算指令
#### TST指令
意思为`test`, 用于按位与运算测试, 但不保存结果, 只更新标志位
**格式**
```assembly
; ===== 示例 1：测试单个位 =====
LDR R0, =GPIOA_IDR          ; 读 GPIO 输入
LDR R0, [R0]
TST R0, #(1 << 5)           ; 测试 PA5 是否为高
BNE pin5_high               ; Z=0 → 引脚为高
BEQ pin5_low                ; Z=1 → 引脚为低

; ===== 示例 2：测试多个位 =====
LDR R0, [R1]                ; 读状态寄存器
TST R0, #0x03               ; 测试 bit0 和 bit1
BNE either_bit_set          ; 任一位为 1 则跳转

; ===== 示例 3：测试所有位是否都为 0 =====
TST R0, #0xFF
BEQ all_zero                ; 低 8 位全为 0

; ===== 示例 4：轮询位 =====
wait_for_flag:
    LDR R0, [R2]            ; 读状态
    TST R0, #0x100          ; 测试 bit8
    BEQ wait_for_flag       ; 为 0 则继续等待

; ===== 示例 5：测试字节的非零 =====
TST R0, #0xFF               ; Z=1 表示低 8 位全 0
BNE has_data

; ===== 示例 6：结合 bit 带操作（Cortex-M3/4） =====
; 如果使用位带区，可以：
LDR R0, =0x22000000         ; bit-band alias
LDR R0, [R0]
TST R0, #1                  ; 测试位
```
#### ORR指令
用于按位或运算
**格式**
```assembly
; 立即数形式
ORR R0, R1, #0xFF         ; R0 = R1 | 0x000000FF

; 寄存器形式
ORR R0, R1, R2            ; R0 = R1 | R2

; 移位形式
ORR R0, R1, R2, LSL #8    ; R0 = R1 | (R2 << 8)

; 特殊：ORR Rd, Rd, #imm（设置 Rd 的某些位）
ORR R0, R0, #(1 << 3)     ; 设置 R0 的 bit3
```

#### BL指令
完成两个动作:
- 把下一条指令的地址存入`LR`寄存器
- 把`PC`寄存器的值改为目标地址

当**嵌套调用函数**时, 因为 **LR 只有一个**，而函数调用可以嵌套。
- 当你在一个函数里 `BL` 调用另一个函数时，**原来的 LR（调用者的返回地址）会被覆盖**。
- 为了防止丢失返回地址，**在被调用函数的入口，几乎总是用 PUSH 把 LR 保存到堆栈里**；函数返回前，再用 POP 把堆栈里的地址恢复到 PC，实现返回。

**格式**
- `BL <symbol>`: 跳转到标号对应的地址执行指令, 并在`LR`中暂存返回地址(即执行跳转指令前的下一条指令的地址)
### 入栈出栈指令

**1. 什么是堆栈（Stack）？**
在程序运行时，内存中有一块专门的区域用作堆栈。它就像一个一端封闭的筒：
- **栈顶（Top）**：唯一能进行数据进出的位置，由一个专门的 **栈指针寄存器**（如 x86 中的 `ESP`/`RSP`）记录地址。
- **生长方向**：大多数架构（x86、ARM 等）都是 **向低地址生长**（压入数据时栈顶地址减小），少数架构相反。
- **生长方式**: 
	- 满生长, 先栈顶指针+1, 再把数据压栈
	- `...`, 先把数据压栈, 然后栈顶指针+1

堆栈主要用于：
- 保存函数返回地址
- 传递函数参数
- 保存寄存器现场（调用者保存或被调用者保存）
- 分配局部变量空间
#### STMFD/STMDB指令
Store Multiple Decrement Before/Full Descending, 即满递减栈存储

**格式**
```assembly
STMFD   r0!, {r4 - r11}         /* push r4 - r11 register */
```
#### PUSH指令
把寄存器的值写入堆栈

**格式**
- `push EBX `: 把EXB寄存器的值写入堆栈
- `push {EBX}`: 上同
#### POP指令
把值弹出到寄存器

**格式**
- `pop EBX` 把栈顶指针上的数据弹出到EBX寄存器
- `pop {EBX}`: 上同

# 常见问题
## 栈溢出
线程栈的大小是固定的（比如 `tshell` 栈为 0x1000 字节）。当函数调用层数过深，或局部变量太大，导致 **SP 指针越过栈底（即最低合法地址）**，再执行 PUSH 或存储器写操作时，就会访问到未分配或受保护的内存
## 栈破坏
栈本身的空间是足够的，但 **某段代码写越界，踩到了栈上已保存的关键数据**（如返回地址、寄存器备份），导致函数返回时跳转到非法地址。

**经典的**
数组的分配, 高地址为数组末端, 低地址为数组前端, 因此从数组末端继续向后写, 就会覆盖高地址之前存储的重要数据, 导致栈被破坏

# 背景
- 芯片的实际启动流程以芯片厂商具体实现为主, 细节查阅芯片厂商针对于每种芯片的**参考手册**
# MCU启动流程
- 只是一个大概的启动流程
## 1. 硬件复位与向量表取址
**主要功能**: CPU读取向量表中的初始栈指针和复位向量, 跳转到复位处理程序
- 上电后, CPU从地址`0x0`读取**初始栈指针(MSP)** 的值, 加载到`SP`寄存器
- 接着从地址`0x4`读取**复位向量(Reset_Handler)** 的地址, 放入`PC`寄存器
硬件自动完成这两步
## 2. 复位处理程序
**主要功能**: 设置必要的环境, 为C/C++运行做准备, 调用系统初始化函数
- 设置栈指针
- 初始化中断向量表偏移 (某些芯片需要重定位向量表到 RAM，但一般默认在 Flash)
- 调用`SystemInit`函数
- 跳转到`_main`(这一步通常由芯片厂商提供的启动文件完成, 开发者一般不需要修改, 但需要了解)
## 3. C/C++运行时环境初始化
**主要功能**: 建立`C/C++`运行时所需的静态内存环境, 包括数据段复制和`BSS`段清零

`__main` 是 ARM 编译器（如 ARMCC、GCC）提供的库函数入口，它执行以下操作
1. **复制 `.data` 段**：将 Flash 中已初始化的全局/静态变量的初值复制到 RAM 中。
2. **清零 `.bss` 段**：将未初始化的全局/静态变量所在的 RAM 区域全部清零。
3. **初始化堆和栈**（如果需要）：
    - 设置堆的起始和结束地址（供 `malloc` 使用）。
    - 某些情况下也会再次设置栈指针（如果之前未正确配置）。
4. **调用 C++ 全局构造函数**（如果使用 C++）。
5. **跳转到用户 `main` 函数**。

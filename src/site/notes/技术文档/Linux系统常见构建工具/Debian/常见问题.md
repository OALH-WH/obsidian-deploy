# 某个版本已经EOF, 继续使用这个版本的官方软件源会出现 `UPDATE`失败
- 由于Debian buster已经EOL，官方的软件仓库已经移动到归档（archive）服务器。通常情况下，当系统生命周期结束后，官方的软件仓库会从主要的镜像中移除，并转移到归档镜像中
## 解决方案
### 用 `snapshot.debian.org`(快照)
- 比 `archive.debian.org`稳定, 包和依赖完整一致
- 注意
	- 需要关闭相关问题的校验
		- 元数据过期
			- 在软件源配置中 `eg. /etc/apt/sources.list`添加选项 `[check-valid-until=no]`, 参考[[技术文档/Linux/linux常用操作/软件包管理/apt包管理/配置\|配置]]
- 示例
```shell
RUN printf "deb http://snapshot.debian.org/archive/debian/20190701T000000Z buster main\n" \
> /etc/apt/sources.list \
 && echo 'Acquire::Check-Valid-Until \"false\";' > /etc/apt/apt.conf.d/99no-check \
 && apt-get update

```

### 用新系统工具, 即拷贝到旧系统
- 注意
	- glibc兼容问题, 新版本软件在旧版本系统中可能用不了
### 手动下载DEB包, 拷贝到旧系统
- 注意
	- 最终还是找软件源网址, 从网址中手动下载
### 临时方案
- 将软件源换到归档仓库中, 即 `deb.debian.org` 替换为 `archive.debian.org`
	- 使用包管理软件, 只能保证可以update, 但是install可能失败
- 示例
```shell
# 原来的行可能是这样的：  
deb http://deb.debian.org/debian buster main

# 改为：  
deb http://archive.debian.org/debian buster main
```

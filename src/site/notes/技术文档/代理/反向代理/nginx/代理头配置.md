# 背景
默认情况下，Nginx 转发请求时会丢失部分客户端原始信息，需要通过`proxy_set_header`指令补充传递


# 常用代理头
## `Host`
`Host` 头告诉后端服务器客户端访问的是哪个域名。Nginx 中有两个相关变量：

| 变量           | 含义                                                                                 |
| ------------ | ---------------------------------------------------------------------------------- |
| `$http_host` | 客户端请求中的原始 Host 值（用户在浏览器输入的域名+端口）[](https://bbs.huaweicloud.com/blogs/438656)       |
| `$host`      | `Nginx` 当前处理的服务器名称（由 `server_name` 决定）[](https://bbs.huaweicloud.com/blogs/438656) |

# `X-Forwarded-For`
当 Nginx 作为代理时，后端服务器默认看到的是 Nginx 的 IP，而不是真实客户端 IP

| 变量                           | 含义                                            |
| ---------------------------- | --------------------------------------------- |
| `$proxy_add_x_forwarded_for` | 会自动将原始 `X-Forwarded-For` 值加上当前客户端 IP，形成完整的代理链 |

## `X-Forwarded-Proto`
当 `Nginx` 处理 `HTTPS` 请求但后端只认识 `HTTP` 时，后端可能无法正确识别请求协议，导致重定向等问题

| 变量                        | 含义                                              |
| ------------------------- | ----------------------------------------------- |
| `$http_x_forwarded_proto` | 告诉后端服务器，客户端最初是用什么协议（`http` 还是 `https`）来访问代理服务器的 |
